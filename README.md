# **Android‑приложение**
## Функции Android‑приложения

- **Главное меню (`MainActivity`)**
  - Переход к калькулятору, плееру, экрану геолокации, работе с сокетами и отправке JSON.

- **Калькулятор (`CalculatorActivity`)**
  - Базовые операции (+, −, ×, ÷) над двумя числами.
  - Отображение выражения и результата на экране.

- **Музыкальный плеер (`PlayerActivity`)**
  - Воспроизведение локального аудиофайла из ресурсов.
  - Кнопки Play / Pause / Stop.

- **Геолокация (`LocationActivity`)**
  - Запрос разрешений на доступ к местоположению.
  - Получение текущих координат через Fused Location Provider.
  - Показ широты, долготы и точности на экране.

## Работа с сокетами (ZMQ)

- **`SocketsActivity`**
  - In‑app demo: эмуляция обмена сообщениями между «клиентом» и «сервером» внутри одной Activity (логика REQ/REP показана в коде, состояние отображается в TextView).
  - Android → PC: реальный клиент ZMQ (REQ), отправляет строку `"Hello from Android!"` на Python‑сервер и получает ответ `"Hello from Server!"`.

- **`server_zmq.py` (режим строкового обмена)**
  - Сервер ZMQ (REP) на Python.
  - Слушает `tcp://*:<PORT>`, принимает строки от Android и отвечает `"Hello from Server!"`.
  - Для проверки используется тестовый REQ‑клиент на ПК; также сервер умеет писать принятые сообщения в файл.

## Сериализация и JSON

- **Data‑классы (`MobileData.kt`)**
  - `LocationDto` – координаты и параметры GPS.
  - `CellInfoLteDto`, `CellInfoGsmDto`, `CellInfoNrDto` – параметры сот LTE/GSM/NR.
  - `MeasurementDto` – общая структура: `location` + один/несколько `CellInfo`.

- **`MobileDataActivity`**
  - Раз в секунду получает:
    - текущую геопозицию (Fused Location);
    - информацию о сотах через `TelephonyManager.allCellInfo`.
  - Заполняет `MeasurementDto` и сериализует его в JSON (Gson).
  - Отправляет JSON на сервер по ZMQ (REQ), показывает статус и время последней успешной отправки.

- **`server_zmq.py` (режим JSON‑лога)**
  - Принимает JSON‑строки от Android.
  - Парсит их и сохраняет в файл `measurements.jsonl` в формате JSON Lines (по одной записи на строку).
  - Логирует номер сообщения и укороченный JSON в консоль.
